## Overview

Log includes a built-in URL shortening service that allows users to create short, shareable links. This feature is useful for sharing posts, collections, or external URLs with custom, memorable slugs.

## What is URL Shortening?

URL shortening transforms long URLs into compact, easy-to-share links:

```
Before: https://yourinstance.com/posts/view/_ITM1a2b3c4d5e6f7g8h9i0j
After:  https://yourinstance.com/s/sunset
```

<Card title="Benefits" icon="link">
  Short URLs are easier to share, remember, and track. They're perfect for social media, printed materials, or anywhere character count matters.
</Card>

## Database Schema

Short URLs are stored in the `url_shorter` table:

```typescript
{
  id: number,              // Auto-incrementing primary key
  urlSlug: string,         // Short URL identifier (unique)
  targetUrl: string,       // Destination URL
  byUser: string,          // User ID reference
  createdAt: Date,
  updatedAt: Date
}
```

### Schema Details

<ParamField path="urlSlug" type="string" required>
  The short identifier used in the URL (e.g., "sunset" in `/s/sunset`). Must be unique across the instance.
</ParamField>

<ParamField path="targetUrl" type="string" required>
  The full destination URL where users will be redirected
</ParamField>

<ParamField path="byUser" type="string" required>
  Reference to the user who created the short link
</ParamField>

## URL Redirection

When users visit a short URL:

1. **Request**: User accesses `/s/[slug]`
2. **Lookup**: System queries the `url_shorter` table for matching slug
3. **Redirect**: User is redirected to the target URL
4. **Tracking**: (Future) Visit statistics are recorded

<Note>
  Redirection is handled server-side with appropriate HTTP 301 (permanent) or 302 (temporary) status codes.
</Note>

## Slug Requirements

Slug creation follows these rules:

<AccordionGroup>
  <Accordion title="Uniqueness">
    Each slug must be unique across the entire instance. Duplicate slugs are rejected.
  </Accordion>
  <Accordion title="Format">
    Slugs should be URL-safe: lowercase letters, numbers, and hyphens only.
  </Accordion>
  <Accordion title="Length">
    Recommended length: 3-20 characters for optimal memorability and uniqueness.
  </Accordion>
  <Accordion title="Reserved Words">
    Certain slugs may be reserved (e.g., "admin", "api", "dashboard") to prevent conflicts.
  </Accordion>
</AccordionGroup>

## Use Cases

### Share Posts Easily

Create memorable links for your posts:

```
Instead of: /posts/_ITM1a2b3c4d5e6f7g8h9i0j
Use:        /s/my-latest-photo
```

### Collection Sharing

Simplify collection URLs:

```
Instead of: /collections/summer-vacation-2024-hawaii-trip
Use:        /s/hawaii
```

### External Links

Shorten and track external links:

```
Instead of: https://example.com/very/long/path/to/resource?param=value
Use:        /s/resource
```

<Card title="Pro Tip" icon="star">
  Use descriptive slugs that hint at the content. "vacation-photos" is better than "abc123".
</Card>

## Managing Short Links

### Viewing Your Links

Users can view all their created short links in a management dashboard:

- List of all short URLs
- Target URL for each link
- Creation date
- Edit and delete options

### Editing Links

Short links can be modified:

- **Update Target**: Change where the link redirects
- **Change Slug**: Modify the short URL identifier (if not in use)
- **Delete**: Remove the short link entirely

<Warning>
  Changing a slug that's already been shared will break existing links. Consider creating a new short URL instead.
</Warning>

## Authentication & Permissions

URL shortening requires authentication:

- Users must be logged in to create short links
- Users can only edit/delete their own links
- Admins may have moderation capabilities

## Security Considerations

<Warning>
  Short URLs can obscure the destination, which may pose security risks. Log implements several safeguards:
</Warning>

- **User Association**: All links are tied to user accounts for accountability
- **Validation**: Target URLs are validated before creation
- **Moderation**: Admins can review and remove malicious links
- **Rate Limiting**: (Planned) Prevent abuse through rate limits

<Tip>
  Users should be cautious when clicking short links from unknown sources, even within trusted platforms.
</Tip>

## Technical Implementation

### Database Constraints

The schema includes important constraints:

```sql
CREATE TABLE url_shorter (
  id INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  url_slug TEXT NOT NULL UNIQUE,  -- Ensures slug uniqueness
  target_url TEXT NOT NULL,
  by_user TEXT NOT NULL REFERENCES user(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

### Index Optimization

For fast lookups:

- Index on `url_slug` for quick slug-to-URL resolution
- Index on `by_user` for efficient user link queries
