## How It Works

When a short URL is accessed:

1. **Request**: User visits `/u/[slug]`
2. **Database Lookup**: System queries `url_shorter` table for the slug
3. **Redirect**: If found, redirects to target URL (HTTP 301)
4. **404 Handling**: If not found, redirects to `/u/404`

From `~/workspace/source/apps/web/src/app/u/[slug]/route.ts:10-27`:

```typescript
export const GET = async (request: NextRequest, context) => {
  const { slug } = await context.params;
  const data = await db
    .select()
    .from(main_schema.urlShorter)
    .where(dorm.eq(main_schema.urlShorter.urlSlug, slug));
  
  if (data.length === 0) {
    return Response.redirect(new URL("/u/404", request.url), 307);
  }
  
  return Response.redirect(new URL(data[0].targetUrl, request.url), 301);
};
```

## Database Schema

Short URLs are stored in the `url_shorter` table:

```sql
CREATE TABLE url_shorter (
  id INTEGER PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  url_slug TEXT NOT NULL UNIQUE,
  target_url TEXT NOT NULL,
  by_user TEXT NOT NULL REFERENCES user(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

## URL Format

Short URLs use the `/u/` prefix:

```
https://yourinstance.com/u/[slug]
```

Examples:
- `https://yourinstance.com/u/sunset` → redirects to target URL
- `https://yourinstance.com/u/project-demo` → redirects to target URL
- `https://yourinstance.com/u/invalid` → redirects to `/u/404`

<Note>
  The slug must be URL-safe (letters, numbers, hyphens, underscores).
</Note>

## Manual Creation (Database)

Until the UI is implemented, administrators can create short links directly in the database:

```sql
INSERT INTO url_shorter (url_slug, target_url, by_user)
VALUES ('my-link', 'https://example.com/full/path', 'user_id_here');
```

<Warning>
  Ensure the `url_slug` is unique and the `by_user` references a valid user ID from the `user` table.
</Warning>

## Use Cases

Once fully implemented, URL shortening will support:

### Share Posts
Transform long post URLs into memorable links:
```
Before: /i/_ITM1a2b3c4d5e6f7g8h9i0j

```

### Collection Sharing
Simplify collection URLs:
```
Before: /c/summer-vacation-2024-hawaii-trip
After:  /u/hawaii
```

### External Links
Shorten external resources:
```
Before: https://example.com/very/long/path?param=value
After:  /u/resource
```

## Technical Details

### Redirect Behavior

- **301 (Permanent Redirect)**: Used for existing short links
- **307 (Temporary Redirect)**: Used when redirecting to 404 page
- **Cache-Control**: Redirects are cacheable by browsers

### Database Constraints

- `url_slug` has a UNIQUE constraint to prevent duplicates
- `by_user` has a foreign key constraint to the `user` table
- Cascade behavior on user deletion is not yet defined
