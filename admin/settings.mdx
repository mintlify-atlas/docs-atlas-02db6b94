## Overview

Log stores system configuration in a key-value (KV) data table, allowing dynamic configuration without code changes. Settings control features like registration, home page display, search functionality, and more.

## KV Data Schema

The settings are stored in the `kv_data` table:

```typescript ~/workspace/source/packages/db/src/schema/main.ts
export const kvData = pgTable("kv_data", {
  id: integer("id").generatedByDefaultAsIdentity().primaryKey(),
  key: text("key").notNull().unique(),
  value: jsonb("value").notNull(),
  createdAt: timestamp("created_at").defaultNow(),
  updatedAt: timestamp("updated_at").defaultNow(),
});
```

<Info>
The `value` field uses JSONB, allowing storage of any JSON-serializable data including booleans, strings, numbers, objects, and arrays.
</Info>

### Registration Control

The registration control is enforced at the database level during user creation:

```typescript ~/workspace/source/packages/auth/src/index.ts
const checkIfSystemDisabledRegister = await db
  .select()
  .from(main_schema.kvData)
  .where(dorm.eq(main_schema.kvData.key, "registrationStatus"))
  .limit(1);

if (checkIfSystemDisabledRegister[0]?.value === false) {
  throw new Error("Registration is disabled");
}
```

<Warning>
Disabling registration prevents all new user signups, but does not affect existing users. The first user registration always succeeds to create the initial admin.
</Warning>

### Update Toggle Settings

```typescript ~/workspace/source/apps/web/src/app/api/data/settings/route.ts
// POST /api/data/settings?tab=settings
await db
  .update(main_schema.kvData)
  .set({ value: body.data.homePage })
  .where(dorm.eq(main_schema.kvData.key, "homePageStatus"));

await db
  .update(main_schema.kvData)
  .set({ value: body.data.registration })
  .where(dorm.eq(main_schema.kvData.key, "registrationStatus"));

await db
  .update(main_schema.kvData)
  .set({ value: body.data.robotsTxt })
  .where(dorm.eq(main_schema.kvData.key, "robotsTxtStatus"));

await db
  .update(main_schema.kvData)
  .set({ value: body.data.search })
  .where(dorm.eq(main_schema.kvData.key, "searchStatus"));

await db
  .update(main_schema.kvData)
  .set({ value: body.data.displayVersion })
  .where(dorm.eq(main_schema.kvData.key, "exposeVersion"));
```

```typescript

type RobotsParsedJson = Record<string, {
```

<CodeGroup>

## Authorization

All settings endpoints require admin authentication:

```typescript ~/workspace/source/apps/web/src/app/api/data/settings/route.ts
if (session.user.role !== "admin") {
  statusCode = 403;
  throw new Error("ERR_USR_INVALID_PERMS");
}
```

<Info>
Only users with the `admin` role can access and modify system settings. Regular users receive a 403 Forbidden error.
</Info>

## Best Practices

<Steps>
  <Step title="Disable Registration After Initial Setup">
    After creating necessary accounts, consider disabling registration to prevent unauthorized signups.
  </Step>
  
  <Step title="Configure Robots.txt Carefully">
    Review your robots.txt rules to ensure search engines can properly index your content.
  </Step>
  
  <Step title="Use Version Display Wisely">
    Only expose version information if you're comfortable with users knowing your Log version.
  </Step>
  
  <Step title="Monitor Search Status">
    Disable search temporarily if you notice performance issues or need to reindex.
  </Step>
</Steps>
