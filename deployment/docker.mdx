Log uses a multi-stage Docker build process with Bun as the runtime. This guide covers building and deploying the application using Docker.

## Prerequisites

- Docker and Docker Buildx installed
- PostgreSQL database (local or remote)
- S3-compatible storage (AWS S3, Cloudflare R2, MinIO, etc.)
- Git for cloning the repository

## Docker Build Process

The Log application uses a two-stage Dockerfile for optimal image size and performance.

### Build Stage

The builder stage installs dependencies and builds the Next.js application:

```dockerfile
FROM oven/bun:latest AS builder
WORKDIR /app
COPY . .
RUN bun install
RUN bun run build
```

### Production Stage

The production stage creates a minimal runtime image:

```dockerfile
FROM oven/bun:latest
WORKDIR /app

ENV NODE_ENV=production
ENV NEXT_TELEMETRY_DISABLED=1

# Copy built application
COPY --from=builder /app/apps/web/.next ./.next
COPY --from=builder /app/apps/web/public ./public
COPY --from=builder /app/apps/web/package.json ./package.json
COPY --from=builder /app/apps/web/next.config.ts ./next.config.ts
COPY --from=builder /app/packages ./packages
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./package.json

EXPOSE 3000
ENV PORT=3000
ENV HOSTNAME="0.0.0.0"

CMD ["bun", "start"]
```

## Building the Image

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/hpware/logs
    cd logs
    ```
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file with your configuration. See the [Environment Variables](/deployment/environment-variables) page for details.

    ```bash
    cp apps/web/.env.example apps/web/.env
    nano apps/web/.env
    ```
  </Step>

  <Step title="Build the Docker image">
    ```bash
    docker build -t logs:latest .
    ```

    For a custom image name:
    ```bash
    docker build -t logs-hosting-custom-image .
    ```
  </Step>
</Steps>

## Running the Container

### Basic Run Command

```bash
docker run -d \
  --name logs \
  -p 3000:3000 \
  --env-file .env \
  logs:latest
```

### With Environment Variables

```bash
docker run -d \
  --name logs \
  -p 3000:3000 \
  -e DATABASE_URL="postgresql://user:pass@host:5432/logs" \
  -e BETTER_AUTH_SECRET="your-secret-key" \
  -e BETTER_AUTH_URL="https://your-domain.com" \
  -e S3_ENDPOINT="https://s3.amazonaws.com" \
  -e S3_ACCESS_KEY_ID="your-access-key" \
  -e S3_SECRET_ACCESS_KEY="your-secret-key" \
  -e S3_BUCKET_NAME="your-bucket" \
  -e S3_REGION="us-east-1" \
  -e CORS_ORIGIN='["https://your-domain.com"]' \
  logs:latest
```

## Database Migration on Startup

The application automatically runs database migrations on startup via the start script:

```json
{
  "scripts": {
    "start": "bun run db:migrate && bun run --filter '*' start"
  }
}
```

This ensures your database schema is always up to date when the container starts.

## Health Checks

The application runs on port 3000 by default. You can add a health check to your Docker setup:

```bash
docker run -d \
  --name logs \
  -p 3000:3000 \
  --env-file .env \
  --health-cmd="curl -f http://localhost:3000 || exit 1" \
  --health-interval=30s \
  --health-timeout=10s \
  --health-retries=3 \
  logs:latest
```

## Docker Compose (Example)

While the repository doesn't include a docker-compose.yml, here's a recommended setup:

```yaml docker-compose.yml
version: '3.8'

services:
  logs:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://logs:password@postgres:5432/logs
      - BETTER_AUTH_SECRET=${BETTER_AUTH_SECRET}
      - BETTER_AUTH_URL=${BETTER_AUTH_URL}
      - S3_ENDPOINT=${S3_ENDPOINT}
      - S3_ACCESS_KEY_ID=${S3_ACCESS_KEY_ID}
      - S3_SECRET_ACCESS_KEY=${S3_SECRET_ACCESS_KEY}
      - S3_BUCKET_NAME=${S3_BUCKET_NAME}
      - S3_REGION=${S3_REGION}
      - CORS_ORIGIN=${CORS_ORIGIN}
    depends_on:
      postgres:
        condition: service_healthy
    restart: unless-stopped

  postgres:
    image: postgres:16-alpine
    environment:
      - POSTGRES_DB=logs
      - POSTGRES_USER=logs
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U logs"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

volumes:
  postgres_data:
```

## Production Considerations

<AccordionGroup>
  <Accordion title="Resource Limits">
    Set memory and CPU limits for production:

    ```bash
    docker run -d \
      --name logs \
      --memory="2g" \
      --cpus="1.5" \
      -p 3000:3000 \
      --env-file .env \
      logs:latest
    ```
  </Accordion>

  <Accordion title="Logging">
    Configure Docker logging drivers for production:

    ```bash
    docker run -d \
      --name logs \
      --log-driver json-file \
      --log-opt max-size=10m \
      --log-opt max-file=3 \
      -p 3000:3000 \
      --env-file .env \
      logs:latest
    ```
  </Accordion>

  <Accordion title="Reverse Proxy">
    Use a reverse proxy like Nginx or Caddy in front of the container:

    ```nginx
    server {
      listen 443 ssl http2;
      server_name your-domain.com;

      ssl_certificate /path/to/cert.pem;
      ssl_certificate_key /path/to/key.pem;

      location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
      }
    }
    ```
  </Accordion>

  <Accordion title="Security">
    - Never expose port 3000 directly in production
    - Use secrets management for sensitive environment variables
    - Run the container with a non-root user if possible
    - Keep the base image updated regularly
    - Use Docker secrets instead of environment variables for sensitive data
  </Accordion>
</AccordionGroup>
